## 📂 Структура памяти по темам

### Правила работы с темами в Telegram

**Каждая тема (тема) в рабочей группе = отдельный проект**

#### Daily notes по темам
- **Файл:** `memory/тема-имя-YYYY-MM-DD.md`
- **Что пишем:**
  - Ход работы по теме
  - Обсуждения и промежуточные результаты
  - Детали, которые нужны для продолжения
  - Все контекстные обсуждения
- **Создаётся:** автоматически при первом сообщении в теме

#### MEMORY.md (долгосрочная память)
- **Что пишем:** только **важное**
  - Финальные решения по теме
  - Важные инсайты и выводы
  - Настройки, которые стоит помнить
  - Ссылки на результаты (если есть)
- **Когда пишем:** когда есть что-то важное, а не каждый шаг
- **Формат:** рубрики по темам (например, "## Тема: Имя темы")

### Пример workflow
1. **Новое сообщение в теме** → создаю/обновляю `memory/тема-имя-YYYY-MM-DD.md`
2. **Обсуждение в теме** → всё фиксирую в daily note
3. **Финальное решение** → кратко записываю в MEMORY.md
4. **Регулярный обзор** → пересматриваю daily notes и обновляю MEMORY.md с важным
