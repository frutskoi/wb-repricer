# Инструкция: Получение доступа к dev.wildberries.ru

## 🎯 Цель

Получить актуальную документацию Wildberries API для репрайсера.

---

## 📋 Шаги

### Шаг 1: Подготовка аккаунта

**Что нужно:**
- ✅ Аккаунт продавца Wildberries (уже есть)
- ✅ API ключ (уже есть в таблице)
- ✅ Доступ к личному кабинету продавца

**Если нет аккаунта продавца:**
1. Зарегистрироваться как продавец на wildberries.ru
2. Верифицировать аккаунт (ИНН, паспорт и т.д.)
3. Добавить хотя бы один товар

---

### Шаг 2: Получение доступа к dev.wildberries.ru

**Способ 1: Через личный кабинет (рекомендуется)**

1. Войти в личный кабинет продавца: https://seller.wildberries.ru/
2. Перейти в раздел "API" или "Разработчикам"
3. Найти ссылку на dev.wildberries.ru или "Документация API"
4. Нажать и получить доступ

**Способ 2: Прямой доступ**

1. Перейти на: https://dev.wildberries.ru/
2. Войти с теми же данными, что в ЛК продавца
3. Пройти антибот проверку (если будет)
4. Получить доступ к OpenAPI документации

**Способ 3: Обратиться в поддержку**

1. Открыть тикет в ЛК продавца: "Помощь → Создать обращение"
2. Тема: "Запрос доступа к API документации"
3. Описание: "Нужен доступ к актуальной документации API на dev.wildberries.ru для интеграции с внешними системами"
4. Приложить: API ключ (для подтверждения)

---

### Шаг 3: Скачивание документации

**Когда доступ получен:**

1. Открыть: https://dev.wildberries.ru/openapi/api-information
2. Найти секцию "Content API" или "Catalog API"
3. Найти эндпоинты для:
   - `GET` или `POST` `/cards/list` (получение товаров)
   - `POST` `/pricing` (получение цен)
   - `POST` `/prices` (загрузка цен)

4. Скачать OpenAPI спецификацию:
   - Найти кнопку "Download" или "Export"
   - Выбрать формат: JSON или YAML
   - Сохранить файл

5. Скопировать примеры запросов:
   - URL эндпоинта
   - Headers (Authorization)
   - Body (формат данных)

---

### Шаг 4: Передача данных КожЗаму

**Что нужно прислать:**

1. **Ссылка на документацию:**
   ```
   https://dev.wildberries.ru/openapi/...
   ```

2. **Или файл OpenAPI:**
   - Прикрепить .json или .yaml файл

3. **Или скриншоты:**
   - Скриншот секции Content API
   - Скриншот примера запроса

4. **Дополнительные данные (если есть):**
   - Рабочий URL для загрузки товаров
   - Формат body запроса
   - Пример успешного ответа

## ⚠️ Возможные проблемы

**Проблема 1: Антибот защита**
- **Решение:** Попробовать через VPN или другой браузер
- **Решение:** Обратиться в поддержку с просьбой дать доступ

**Проблема 2: Нет доступа даже в ЛК продавца**
- **Решение:** Создать тикет в поддержке
- **Тема:** "Запрос доступа к API документации для разработчика"

**Проблема 3: Документация устарела**
- **Решение:** Проверить дату обновления в документации
- **Решение:** Обратиться в поддержку с вопросом об актуальных эндпоинтах

**Проблема 4: Не могу найти нужные эндпоинты**
- **Решение:** Ищите по ключевым словам: "cards", "products", "catalog", "pricing"
- **Решение:** Спросить в поддержку: "Какие endpoint-ы для загрузки товаров и цен?"
