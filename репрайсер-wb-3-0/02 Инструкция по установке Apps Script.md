# Этап 2: Установка Apps Script для Репрайсер WB 3.0

## 📋 Шаг 1: Открытие редактора Apps Script

1. Открой [таблицу](https://docs.google.com/spreadsheets/d/1BOEIbmDrP4kfrps2Nv1hO1nfa4aFI76waobb9CzR29k/edit)
2. В верхнем меню выбери **Расширения** → **Apps Script**
3. Откроется новая вкладка с редактором кода

## 📋 Шаг 2: Установка кода

### Вариант 1: Скопировать из файла (простой)
1. Открой файл `Code.gs` из проекта (я его создал)
2. Скопируй весь код
3. В редакторе Apps Script найди файл `Code.gs` (уже есть по умолчанию)
4. Удали весь старый код в нём
5. Вставь новый код
6. Нажми **Сохранить** (Ctrl+S)

### Вариант 2: Создать новый файл
1. В редакторе Apps Script нажми **+** рядом с "Файлы"
2. Выбери "Скрипт"
3. Назови файл: `Code.gs`
4. Скопируй код из файла `Code.gs`
5. Вставь в новый файл
6. Нажми **Сохранить**

## 📋 Шаг 3: Настройка авторизации

### Включение Google API
1. В редакторе Apps Script нажми на шестерёнку ⚙️ (настройки проекта)
2. Открой раздел "Google API"
3. Нажми на ссылку "Google API"
4. Откроется Cloud Console для твоего проекта
5. Включи следующие API:
   - **Google Sheets API**
   - **UrlFetchApp** (включен по умолчанию)

Если не открывается:
- В редакторе Apps Script нажми **Редактировать** → **Текущие триггеры проекта**
- Ссылка на проект будет в URL

## 📋 Шаг 4: Первая инициализация

### Запуск onOpen (создание меню)
1. В редакторе Apps Script выбери функцию `onOpen` из выпадающего списка функций
2. Нажми **Запустить** ▶️
3. Дай разрешения:
   - Выбери свой аккаунт Google
   - Разреши доступ к таблицам
   - Если появится предупреждение "Google hasn't verified this app" — нажми **Дополнительно** → **Перейти к ... (безопасно)**
4. После успешного запуска вернись в таблицу
5. В верхнем меню должен появиться новый пункт **Репрайсер WB**

### Запуск onInstall (альтернативный способ)
1. В редакторе Apps Script выбери функцию `onInstall`
2. Нажми **Запустить** ▶️
3. Дай разрешения
4. Меню появится автоматически

## 📋 Шаг 5: Настройка параметров

### Настройка в таблице (предпочтительный способ)
1. Перейди на лист **Настройки** в таблице
2. Заполни следующие параметры:

| Параметр | Описание | Пример |
|----------|----------|--------|
| API ключ WB (только чтение) | Для загрузки товаров и цен | `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...` |
| API ключ WB (чтение+запись) | Для загрузки цен в ЛК | `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...` |
| Токен авторизованного пользователя WB | Для парсинга цен с сайта | `wb_session_value=...` |
| Частота автообновления | Число | `30` |
| Тип частоты | Выбор частоты | `минуты (30) / часы (1) / 2 часа (2)` |
| Пороговая разница цен (руб) | Порог для обновления | `5` |
| Email для уведомлений | Для получения уведомлений | `email@example.com` |

### Где получить токены?

#### API ключи WB
1. Зайди в [Личный кабинет WB](https://seller.wildberries.ru/)
2. Настройки → Разработчикам → Доступ к API
3. Создай новые токены:
   - Токен 1: права "Только чтение"
   - Токен 2: права "Чтение + Запись"
4. Скопируй токены в настройки

#### Токен авторизованного пользователя (для парсинга)
1. Открой сайт [wildberries.ru](https://www.wildberries.ru)
2. Зайди в свой аккаунт
3. Открой DevTools (F12) → вкладка **Application** (или **Storage**)
4. Разверни **Cookies** → `https://www.wildberries.ru`
5. Найди куки `wb_session_value`
6. Скопируй значение в настройки

Важно: Токен авторизованного пользователя имеет ограниченный срок действия (несколько дней). Нужно обновлять его периодически.

## 📋 Шаг 6: Синхронизация настроек

1. В таблице выбери меню **Репрайсер WB** → **📊 Синхронизировать настройки**
2. Настройки сохранятся в PropertiesService

## 📋 Шаг 7: Тестирование функционала

### Тест 1: Загрузка товаров
1. В меню **Репрайсер WB** выбери **📥 Загрузить товары из WB**
2. Дождись завершения
3. Проверь лист **Товары** — должны появиться товары из WB

### Тест 2: Парсинг цен
1. В меню **Репрайсер WB** выбери **🌐 Парсить цены с сайта WB**
2. Дождись завершения
3. Проверь столбцы 11, 12, 13, 14 — должны появиться цены

### Тест 3: Расчёт цен
1. Заполни столбцы 6 (РРЦ) и 7 (Модель удержания) для нескольких товаров
2. В меню **Репрайсер WB** выбери **🧮 Рассчитать цены для загрузки**
3. Проверь столбец 15 (Загружаемая цена)

### Тест 4: Загрузка цен в ЛК
1. В меню **Репрайсер WB** выбери **📤 Загрузить цены в ЛК WB**
2. Дождись завершения
3. Проверь столбец 16 (Статус загрузки)
4. Проверь лист **Лог** — там будет история операций

### Тест 5: Полное обновление
1. В меню **Репрайсер WB** выбери **🔄 Полное обновление**
2. Выполнит все шаги автоматически: парсинг → расчёт → загрузка

## 📋 Шаг 8: Настройка автообновления

1. Заполни настройки в таблице (частота обновления)
2. В меню **Репрайсер WB** выбери **⏰ Настроить автообновление**
3. Получишь сообщение о настройке триггера
4. Репрайсер будет автоматически обновляться каждые X минут/часов

Для остановки автообновления:
1. В меню **Репрайсер WB** выбери **🚫 Остановить автообновление**

## 📋 Шаг 9: Проверка логов

1. Перейди на лист **Лог**
2. Там будет история всех операций:
   - Дата и время
   - Артикул WB
   - Действие
   - Старая цена
   - Новая цена
   - Статус
   - Сообщение

## 🔧 Расширенные настройки

### Изменение частоты автообновления
1. Измени значение в настройках таблицы
2. Выбери **Репрайсер WB** → **⏰ Настроить автообновление**

### Изменение параметров расчёта
Код открыт для редактирования. Можешь изменить логику в функции `calculateUploadPrice`:
```javascript
function calculateUploadPrice(rrp, wbDiscount, walletPercent, model) {
  // Твоя логика здесь
}
```

### Добавление новых функций
Добавляй новые функции в файл `Code.gs` и вызывай их через меню.

## 🚨 Возможные проблемы

### "Google hasn't verified this app"
1. Нажми **Дополнительно**
2. Нажми **Перейти к ... (безопасно)**
3. Разреши доступ

### Ошибка "API Error 401: Unauthorized"
- Проверь API ключи в настройках
- Убедись, что ключи действительны

### Ошибка "API Error 429: Too Many Requests"
- Снизь частоту запросов
- Увеличь паузы между запросами в коде (Utilities.sleep)

### Парсинг не работает
- Проверь токен авторизованного пользователя
- Возможно, токен устарел — обнови его

## 📞 Поддержка

Если возникнут проблемы:
1. Проверь логи на листе **Лог**
2. Проверь настройки на листе **Настройки**
3. Проверь код в редакторе Apps Script
